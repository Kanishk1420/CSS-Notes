Chapter 5 - Size, Position & Lists

  • A Responsive Website gives great user experience on any device. Ex: iPad, iPhone, Android, etc.

  • There are more units for describing size other than px, like vh, vw, em, rem, %, etc.

    • What's wrong with pixels? These units are relative to the other length property. Following are some of the most commonly used relative lengths.

  • Relative lengths: These units are relative to the other property.
  • Following are some of the most commonly used relative lengths:
    • em: unit relative to the parent font size.
    • rem: unit relative to the root font size.
    • vw: unit relative to 1% viewport width.
    • vh: unit relative to 1% viewport height.
    • %: unit relative to parent element.

Min/Max Height & Width

  • If the content is smaller (e.g., your height and width are already set as 4px, vh, or vw), then if you set min-width as 50px, the height and width will not change to 4px; it will remain at 50px by default.
  • If you set the width and height as 5px, vh, or vw, and max-width as 50px, then the width and height will be by default as 5px, but if you choose height and width as 100px or vw, then it will remain at 50px by default.

The Position Property

    Used to manulipate the location of an element Following are the possible values.

  • Static:-The default position (Top/bottom/left/Right/Z-index has no effect).

  • Relative:-(The top/bottom/left/right/z-index will now work).otherwise the element is in the flow of document like static.

  • Absolute:-The Element is removed from the flow & is relativity positioned to its first non-static ancestor. top/bottom works.

  • Fixed:-just like absolute except the element is positioned relative to browser window.

  • Sticky:-The element is positioned based on user scroll position.

    • Z-index defines which layer will be above which in case is of overlapping elements.
